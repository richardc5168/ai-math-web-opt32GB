# Parent View MVP

## Required Panels

1. Child summary
2. Mastered content
3. Needs strengthening
4. Remediation suggestions
5. Pre/post trend
6. Parent action guidance

## Constraints

- Parent sees exactly one child context at a time.
- Parent never sees class aggregate.
- Parent-readable wording preferred over internal error-code jargon.

## Evidence Requirements

- Show child's own data only
- Show source period or assessment ids behind the summary
- Explain uncertainty when pre/post uses different but equivalent questions
